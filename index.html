Git

Pasos para su uso

  1. Descargar Git desde la página de Git
  2. Lo instalamos
  3. Despues de instalado, abrimos la terminal de git bash
  4. Para probar que esta instalado, colocamos en la terminal de comandos git --version

Una vez dentro de git lo primero que hacemos es configurar de forma global los parametros básicos

  1. Colocamos nuesto correo con el comando git config --global user.gmail nombre@email.com
  2. Agregamos nuestro nombre git config --global user.name Pepito o si vas a agragar tu nombre completo con espacios, colocas "Pepito Perez"
  3. Para que se muetren colores en la consola, ponemos git config --global color.ui true

Despuesde haber realizado estas configuraciones basicas, iniciamos con los comandos

Comandos de la terminal:

  • cd Cambiar de directorio
  • cd.. Salir de directorio
  • mkdir Crear carpeta
  • ls Listar directorios
  • clear Limpiar la terminal
  • touch nombreArchivo.extension Crear archivos
  • rm nombreArchivo.extension Borrar archivos
  • rm -rf nombreCarpeta Borrar carpeta

Antes de ver los comandos de Git debemos conocer los tres estados de git

  1. Working area: es el area de trabajo en la que estamos realizando los cambios
  2. Staging area: este estado es donde los archivos estan agregados o con los cambios realizados antes de pasar al ultimo estado
  3. Git directory: estado final donde almacenara el proyecto con todo todos los metadatos, es la parte mas importante de git

Comandos de Git

  • git init permite iniciar un repositorio en el directorio en el que estamos
  • Una vez creado nuestro proyecto los archivos estaran en un estado untraked es decir que no estan en el repositorio sino en el directorio de trabajo, para ver el estado de los archivos ejecutamos el comando git status
  • git add nombreArchivo para agregar nuestros cambios al siguiente estado usamos para agregar un archivo especifico, si queremos agregar todo de una sola vez usamos git add -A
  • Si agregamos un archivo que por algun motivo no queriamos agregar podemos deshacer los cambios con git rm --cached nombreArchivo
  • git rm -f nombreArchivo sirve para eliminar completamente un archivo
    • COMMITS
  • git commit -m "Mensaje" permite enviar los archivos que esten en STAGING AREA al GIT DIRECTORY
  • git commit --amend permite agregar cambios al ultimo commit
    • TAGS ó Versiones
  • git tag -a [numero version] -m 'mensajeVersion'Sirve para agregar la version de nuestro proyecto
  • git tag -lver la version
  • git tag -fpara renombrar
  • git tag -d [numero version]Elimina el tag en el commit
    • Configurar log
  • git log --oneline Permite que se muestren los logs en una sola linea de forma compacta
  • git log --graph Mustra el grafico de las ramas dentro del arbol de una manera mas comoda
  • git log -[numero] Muestra la cantidad de log que necesitemos

    • Comparto la configuración que uso: git config --global alias.superlog "log --graph --abbrev-commit --date=relative --all --format=format:'%C(bold blue)%h%C(reset) %C(auto,magenta)%G? %C(bold green)(%>(18,trunc)%aD)%C(reset) %C(bold blue)[% <(10,trunc)%aN]%Creset - %s%C(reset) %C(bold yellow)%d%C(reset) '"

    DIFF
  • git diffMuestra los cambios realizados en dos commit diferentes, se puede usar solo o colocando los SHA o versiones(tags) de cada commit
    • RESET
  • git reset --soft [SHA] Quita un cambio del commit y lo regresa al staging directory
  • git reset --mixed [SHA] Quita cambios del commit y los regresa al working directory
  • git reset --hard [SHA] Elimina los cambios incluso del working directory, es el más peligroso de todos porque podemos perder parte de nuestro trabajo
    • BRANCH o Ramas
  • git branch [nombreBranch] Crea una nueva rama
  • git branch -l Permite listar las ramas existentes
  • git branch -d [nombreBranch] Elimina la rama, en caso de que no tenga cambios o commits
  • git branch -D [nombreBranch] Elimina la rama de forma forza sin importar si tiene commits
  • git branch -m [nombreActual] [nuevoNombre] Editar el nombre de una rama
  • git checkout [nombreRama ó SHA del commit] Sirve para movernos entre ramas o commits para crear nuevas ramas a partir de ese commit
  • git checkout -b [nombreBranch] Este comando permite crear una rama y movernos a esa rama creada
    • MERGE
  • git merge [nombreRama] En este comando debemos estar en la rama en la que queremos que se combinen los cambios que se realizaron en las demas ramas Ejemplo: si mi rama principal es master y quiero unir los cambios de otra rama, me ubico en la rama principal con el comando git checkout master una vez estemos en nuetra rama usamos merge y el nombre de la rama que queremos unir, esto creara un nuevo commit con la union de las ramas
    • STASH
  • git stash realiza un guardado temporal en donde no quedara en el staging area ni en un commit en caso de que tengamos que cambiarnos de rama pero aun no queremos hacer un commit, una especie de limbo de cambios donde podremos recuperarlos después
  • git stash list Nos permite listar los stash que tengamos
  • git stash drop stash@{numero} Permite eliminar un stash dentro de nuestra lista
  • git stash apply Este comando recupera el stash realizado a nuestro working directory
    • CHERRY-PICK
  • git cherry-pick [SHA] El comando cherry-pick permite llevar un commit realizado en una rama equivocada a la rama correcta, para esto copiamos el codigo del commit(SHA) que queremos cambiar a otra rama, nos movemos a la rama a la cual queremos mover este commit y ejecutamos este comando.
